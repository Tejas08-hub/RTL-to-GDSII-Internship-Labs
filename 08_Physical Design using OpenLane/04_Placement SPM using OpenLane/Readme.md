# 04_Placement SPM using OpenLane

## Overview

This lab demonstrates the **Placement stage** of the RTL-to-GDSII physical design flow using the **OpenLane** automated ASIC design flow. The synthesized netlist of the **Serial Parallel Multiplier (SPM)** is placed onto the standard cell rows generated during floorplanning.

Placement determines the physical location of each standard cell while minimizing wirelength, reducing congestion, and improving timing performance. At the end of this stage, every standard cell is legally positioned within the core area, making the design ready for clock tree synthesis (CTS) and routing.

---

## Objectives

- Understand the placement stage in the ASIC physical design flow.
- Perform automated placement using OpenLane.

- Prepare the design for the routing stage.

## Design Used

**Design Name:** Serial Parallel Multiplier (SPM)

The Serial Parallel Multiplier is a 32-bit arithmetic circuit that performs multiplication using Carry Save Adders (CSA) and Transition Comparator (TCMP) blocks. During placement, the synthesized logic is converted into physical standard cells and positioned within the core area.

---

## Placement Theory

Placement is the process of assigning an optimized physical location to every standard cell inside the predefined core area created during floorplanning.

The placement engine attempts to:

- Minimize total wirelength.
- Reduce routing congestion.
- Improve timing performance.
- Maintain legal placement without overlapping cells.
- Optimize utilization of the available core area.

The placement stage generally consists of:

1. Global Placement
2. Placement Optimization
3. Detailed Placement
4. Legalization

The output generated after placement serves as the input for Clock Tree Synthesis (CTS) and Routing.
