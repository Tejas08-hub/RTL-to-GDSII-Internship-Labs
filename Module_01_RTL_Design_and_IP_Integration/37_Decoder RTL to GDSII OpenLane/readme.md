# 37. Decoder RTL to GDSII using OpenLane

## Overview

This project demonstrates the complete **RTL-to-GDSII ASIC design flow** using **OpenLane**, an open-source digital ASIC implementation framework. A **4-to-16 Decoder** is designed in Verilog HDL and implemented through the complete physical design flow, including synthesis, floorplanning, placement, clock tree synthesis, routing, layout generation, DRC verification, and LVS verification.

OpenLane automates the complete RTL-to-GDSII flow by integrating industry-standard open-source EDA tools such as **Yosys**, **OpenROAD**, **Magic**, **Netgen**, and **OpenSTA**, enabling the generation of manufacturing-ready GDSII layouts.

# RTL Design

The decoder accepts a **4-bit binary input** and activates exactly one of the sixteen outputs corresponding to the binary input value.

### Inputs

- 4-bit input (`in[3:0]`)

### Outputs

- 16-bit decoded output (`out[15:0]`)

The design is implemented using a combinational `always @(*)` block with a `case` statement to generate the one-hot output.

# RTL-to-GDSII Design Flow

## 1. RTL Design

The Verilog RTL describes the functionality of the decoder circuit.

↓

## 2. Logic Synthesis

OpenLane invokes **Yosys** to:

- Parse RTL
- Optimize logic
- Map logic to SKY130 standard cells
- Generate synthesized gate-level netlist

↓

## 3. Floorplanning

The floorplanner:

- Defines die area
- Allocates core area
- Places IO pins
- Creates power distribution planning

↓

## 4. Placement

OpenROAD performs global and detailed placement by positioning standard cells while minimizing congestion and wirelength.

↓

## 5. Clock Tree Synthesis (CTS)

CTS constructs a balanced clock network to minimize skew and insertion delay.

For this combinational decoder design, CTS has minimal impact because the circuit contains no sequential elements.

↓

## 6. Routing

Global and detailed routing connect all placed standard cells using available metal routing layers.

↓

## 7. Layout Generation

Magic converts the routed DEF into:

- GDSII Layout
- MAGIC Layout
- LEF Views

↓

## 8. DRC Verification

Magic checks the layout against SKY130 design rules to ensure manufacturability.

↓

## 9. LVS Verification

Netgen compares the extracted layout netlist with the synthesized Verilog netlist to ensure logical equivalence.
