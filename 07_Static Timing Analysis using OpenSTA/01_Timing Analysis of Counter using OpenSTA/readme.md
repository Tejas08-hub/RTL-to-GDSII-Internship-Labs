# Static Timing Analysis of 4-Bit Counter using OpenSTA

## Overview

This project demonstrates **Static Timing Analysis (STA)** of a synthesized **4-bit synchronous counter** using **OpenSTA** and the **OSU018 Standard Cell Library**. The RTL design is synthesized using **Yosys**, and the generated gate-level netlist is analyzed under different timing constraints to evaluate setup and hold timing.

The objective of this lab is to understand the complete timing verification flow from synthesis to timing analysis and to interpret timing reports generated by OpenSTA.

---

## Objectives

- Design a 4-bit synchronous counter in Verilog HDL.
- Synthesize the RTL design using Yosys.
- Generate a gate-level netlist.
- Apply timing constraints using an SDC (Synopsys Design Constraints) file.
- Perform Static Timing Analysis using OpenSTA.
- Analyze setup and hold timing paths.
- Compare timing results for different clock periods (10 ns and 5 ns).

## RTL Design

The counter increments on every positive edge of the clock.

```verilog
module counter(
    input clk,
    input rst,
    output reg [3:0] out
);

always @(posedge clk)
begin
    if(!rst)
        out <= 4'b0000;
    else
        out <= out + 1'b1;
end

endmodule
```

---

## Synthesis Flow

The RTL is synthesized using **Yosys** with the OSU018 standard cell library.

### Yosys Script

```
read_verilog counter.v

synth -top counter

dfflibmap -liberty osu018_stdcells.lib

abc -liberty osu018_stdcells.lib

write_verilog counter_synth.v
```

The synthesized gate-level netlist (`counter_synth.v`) is used as the input for OpenSTA.

## Timing Constraints (SDC)

The following timing constraints are applied:

- Output Delay

Clock periods analyzed:

- **10 ns**
- **5 ns**

---

## OpenSTA Flow

### Load Standard Cell Library

```
read_liberty osu018_stdcells.lib
```

### Read Synthesized Netlist

```
read_verilog counter_synth.v
```

### Link Top Module

```
link_design counter
```

### Load Timing Constraints

```
read_sdc counter.sdc
```

### Generate Setup Report

```
report_checks -path_delay max
```

### Generate Hold Report

```
report_checks -path_delay min
```
